# Orakle Backend — Deployment Guide

## Overview

Orakle is a blockchain intelligence platform providing:
- Ethereum wallet risk analysis
- Smart contract security auditing
- Transaction translation
- Solidity file upload scanning
- AI-powered reasoning (via OpenRouter/Gemma)
- PDF report generation
- Solana wallet intelligence
- Dashboard metrics

**Stack:** Django 5.x · PostgreSQL (Supabase) · Gunicorn · WhiteNoise · OpenRouter AI

---

## Local Development Setup

### Prerequisites

- Python 3.11+
- pip
- PostgreSQL (or Supabase account)

### 1. Clone & Install

```bash
cd backend
pip install -r requirements.txt
```

### 2. Environment Variables

Create a `.env` file in the `backend/` directory:

```env
DEBUG=True
SECRET_KEY=django-insecure-your-dev-key-here
DATABASE_URL=postgresql://user:password@host:port/dbname?sslmode=require
ETHERSCAN_API_KEY=your_etherscan_api_key
GEMINI_API_KEY=your_gemini_api_key
ALLOWED_HOSTS=localhost,127.0.0.1
CORS_ALLOWED_ORIGINS=http://localhost:3000,http://127.0.0.1:3000
```

### 3. Database Setup

```bash
python manage.py migrate
```

### 4. Collect Static Files

```bash
python manage.py collectstatic --noinput
```

### 5. Run Development Server

```bash
python manage.py runserver
```

Server starts at `http://127.0.0.1:8000`.

## Third-Party Service Setup

### Supabase (PostgreSQL)

1. Create a project at [supabase.com](https://supabase.com)
2. Go to **Settings → Database → Connection String**
3. Copy the **URI** format connection string
4. Set as `DATABASE_URL` in your environment
5. Ensure `?sslmode=require` is appended

### Etherscan API

1. Register at [etherscan.io](https://etherscan.io/register)
2. Go to **API Keys** → Create new key
3. Set as `ETHERSCAN_API_KEY` in your environment
4. Free tier supports 5 calls/second

### Google Gemini API

1. Get an API key from [Google AI Studio](https://aistudio.google.com/)
2. Set as `GEMINI_API_KEY` in your environment
3. Orakle uses the `gemma-4-26b-a4b-it` model
4. If the key is missing or invalid, AI features gracefully return fallback text

## Render Deployment

### Option A: Using render.yaml (Recommended)

1. Push the repository to GitHub
2. Connect the repository to Render
3. Render auto-detects `render.yaml` and configures the service
4. Set the following environment variables in the Render dashboard:

| Variable | Value |
|----------|-------|
| `DEBUG` | `False` |
| `SECRET_KEY` | *(auto-generated by Render)* |
| `DATABASE_URL` | Your Supabase connection string |
| `ETHERSCAN_API_KEY` | Your Etherscan API key |
| `GEMINI_API_KEY` | Your Gemini API key |
| `ALLOWED_HOSTS` | `.onrender.com` |
| `CORS_ALLOWED_ORIGINS` | Your frontend URL |

### Option B: Manual Setup

1. Create a new **Web Service** on Render
2. Connect your GitHub repository
3. Set the root directory to `backend`
4. Configure:

**Build Command:**
```bash
pip install -r requirements.txt && python manage.py collectstatic --noinput && python manage.py migrate --noinput
```

**Start Command:**
```bash
gunicorn config.wsgi:application --bind 0.0.0.0:$PORT --workers 3 --timeout 120
```

5. Set all environment variables listed above

- [x] HTTPS redirect (`SECURE_SSL_REDIRECT`)
- [x] Secure session cookies (`SESSION_COOKIE_SECURE`)
- [x] Secure CSRF cookies (`CSRF_COOKIE_SECURE`)

- [x] HSTS subdomains (`SECURE_HSTS_INCLUDE_SUBDOMAINS`)

## Troubleshooting

### AI returns "temporarily unavailable"

- Check that `GEMINI_API_KEY` is set and valid
- Check server logs for `401 Unauthorized` errors
- Verify your Google AI Studio account has credits/quota

### Etherscan returns errors

- Verify `ETHERSCAN_API_KEY` is set
- Check rate limits (5 req/s on free tier)
- Etherscan v2 API requires `chainid=1` (set automatically)

### Database connection fails

- Verify `DATABASE_URL` format includes `?sslmode=require`
- Check Supabase project is active
- Ensure connection pooler port (6543) is used

### Static files not loading

- Run `python manage.py collectstatic --noinput`
- Ensure WhiteNoise is in MIDDLEWARE
- Check `STATIC_ROOT` points to `staticfiles/`
